Design Approval

Token Optimization for Obsidian MCP

1. New Parameters

Add to read_note, search_notes, list_notes:

  • mode: "full" | "content" | "summary" (default: "full")
  • fields: string[] - which fields to return (title, content, tags, date, status, related)

2. Mode Behavior

  • full (default): Returns raw markdown with frontmatter (current behavior)
  • content: Returns only the body content, no frontmatter
  • summary: Returns just title, tags, date, status - no body content

3. Fields Parameter

Allows granular control - useful for list operations where you need titles but not content:

  • fields: ["title", "tags"] - lightweight for dropdowns/dashboards
  • fields: ["title", "content"] - exclude frontmatter fields

4. Backward Compatibility

Default is "full" so existing callers work unchanged. Tools like create_note, update_note unaffected.
