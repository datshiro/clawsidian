Token Optimization Approaches

Choose how much content to return from MCP tools

A

Minimal Response Mode

Return only content + frontmatter, not full raw markdown

~60% token reduction

B

Content-Only Mode

Option to exclude frontmatter entirely - body text only

~80% token reduction

C

Selective Fields

Let caller specify which fields to return (title, content, tags, etc.)

Most flexible but more complex

D

Result Caching

Cache recent reads to avoid re-fetching same notes

Memory trade-off, risk of stale data

Recommended: A + B + C

Add a mode parameter to read_note, search_notes, and list_notes that lets callers choose:

  • full (default): current behavior
  • content: body only, no frontmatter
  • summary: just title + tags
